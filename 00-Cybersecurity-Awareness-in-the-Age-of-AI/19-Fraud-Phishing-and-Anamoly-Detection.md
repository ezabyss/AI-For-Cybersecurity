# 🤖 AI in Cybersecurity: Fraud, Phishing & Anomaly Detection

---

# 📌 1. Big Picture

👉 Organizations generate:

- Emails 📧  
- Logins 🔐  
- Transactions 💳  
- File activity 📂  

---

👉 Problem:

**Too much noise**

---

## ⚠️ Key Idea

👉 Threats hide inside normal activity  

---

# 🧠 2. Core Idea

👉 **AI detects threats by learning normal behavior and spotting what doesn’t belong**

---

# 🔍 3. Traditional vs AI Detection

| Traditional Security | AI-Based Security |
|--------------------|------------------|
| Static rules | Dynamic learning |
| Fixed thresholds | Behavior-based |
| Slow adaptation | Real-time response |
| Limited detection | Pattern + anomaly detection |

---

# 💳 4. AI in Fraud Detection

## 🧠 How It Works

👉 AI learns:

- User behavior  
- Transaction patterns  
- Login habits  

---

## ⚠️ Example

👉 Normal:
- User pays regular vendor  

👉 Suspicious:
- New vendor + large transfer  

---

👉 AI detects:
**Deviation from normal**

---

# 📧 5. AI in Phishing Detection

## 🔍 What AI Analyzes

- Language tone  
- Urgency  
- Writing style  
- Domain similarity  

---

## 🧠 Example

👉 Email says:
"Urgent transfer needed now!"

---

👉 AI checks:

- Emotional manipulation  
- Unusual tone  
- Slight domain mismatch  

---

👉 Result:
**Flag / quarantine email**

---

# 🌐 6. AI in Anomaly Detection

👉 AI monitors:

- Logins  
- File access  
- Device connections  
- User activity  

---

## 🧠 Example

👉 User behavior:

- Logs in from Nepal normally  

👉 Suddenly:

- Login from another country 🌍  

---

👉 AI detects:
**Anomaly**

---

# 🔁 7. How AI Thinks

👉 Not just event  

👉 But:
**Pattern + deviation**

---

👉 It asks:

- Is this normal?  
- Does it match past behavior?  
- Is something slightly off?
